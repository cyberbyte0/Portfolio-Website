Security Research

A technical breakdown of my methodology for API security, Android application analysis, and ADB-based automation.

My Research Methodology

I employ a multi-layered approach to security analysis, shifting from high-level reconnaissance to deep logic-flow inspection.

01

Reconnaissance & Mapping

Discovery of hidden endpoints, undocumented parameters, and API versioning (e.g., `/v1/` vs `/v2/`). I use proxy tools to intercept mobile and web traffic simultaneously.

02

Authentication & State Analysis

Testing JWT integrity, token expiration, and session fixation. I analyze how the application handles state across different user roles.

03

Logical Flow Manipulation

Focusing on the OWASP API Top 10, specifically targeting BOLA (Broken Object Level Authorization) and Mass Assignment by fuzzing ID parameters and request bodies.

04

Automated Stress Testing

Developing custom Playwright scripts to simulate race conditions or brute-force scenarios that manual testing might miss.

Technical Focus

Research Interests & Focus Areas

In my independent research and offensive security labs, I focus on identifying critical logic gaps in modern web architectures. My work emphasizes the following vulnerability classes:

  • Authorization Logic (IDOR/BOLA): Analyzing how APIs validate resource ownership to prevent unauthorized data access between users.
  • Information Disclosure: Identifying excessive data exposure in API responses where sensitive PII or system metadata is leaked.
  • Automated Bypass Techniques: Researching ways to simulate realistic user behavior to test the resilience of rate-limits and CAPTCHA implementations.
  • Client-Side Security: Reversing Android applications to inspect SSL pinning, root detection, and local data encryption.
